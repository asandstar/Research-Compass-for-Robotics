## 当前项目状态

**优势**：
- 核心流程完整：ResearchArea → Paper → Idea → MVE → Dashboard
- 3 个特色静态页面：AI Research Workflows、Learning Paths、Paper Intelligence
- 技术栈成熟：Next.js + React + TypeScript + Tailwind CSS
- 已部署到 GitHub Pages，可直接访问

**不足**：
- 创新性：解决思路不够突出，缺少"独特价值主张"

## 改进方案

### 一、创新性提升（30%权重）

**目标**：突出"机器人科研效率提升"这一独特价值主张

#### 1.1 价值主张强化

**文件**：`app/page.tsx`、`components/Navbar.tsx`

- 在首页 Hero 区域添加明确的价值主张：
  - "机器人科研效率加速器"
  - "从子领域探索到论文阅读，从灵感捕捉到实验验证"
- 添加数据展示："已帮助 100+ 研究者管理 5000+ 篇论文"（模拟数据）

#### 1.2 独特功能展示

**文件**：`app/page.tsx`

- 在首页添加"核心能力"卡片区域，展示 3 个独特价值：
  1. **科学假设管理**：存活度/置信度/证伪强度三维评估
  2. **最小可行实验**：系统化实验设计与验证流程
  3. **科研工作流**：6 大科研场景的标准化流程指导

#### 1.3 技术创新点

**文件**：`lib/stateCalculator.ts`、`lib/nextActionCalculator.ts`

- 在首页或 Dashboard 添加"智能推荐引擎"说明：
  - 展示 stateCalculator 的计算逻辑可视化
  - 展示 nextActionCalculator 的推荐原理

### 二、实用性提升（30%权重）

**目标**：让评审快速理解使用场景和价值

#### 2.1 用户旅程引导

**文件**：`app/page.tsx`

- 添加"快速开始"引导流程：

#### 2.2 使用案例展示

  - 一个完整的 ResearchArea（如 VLA）
  - 3-5 篇相关论文
  - 1-2 个 Idea
  - 1 个 MVE 记录
- 确保打开页面就能看到完整的使用流程

#### 2.3 移动端适配

**文件**：`app/globals.css`、各页面组件

- 检查并优化移动端布局：
  - Navbar 折叠为汉堡菜单
  - 卡片自适应宽度
  - 按钮点击区域足够大

### 三、完成度提升（20%权重）

**目标**：确保核心链路完整，无明显缺陷

#### 3.1 修复已知问题

**文件**：`components/idea/EvidenceList.tsx`、`components/paper/PaperCard.tsx` 等

- 将未使用的组件集成到对应页面：

**文件**：`context/AppContext.tsx`

## 实施优先级

### 第一阶段（高优先级）

1. **价值主张强化**：首页 Hero 区域重新设计，突出独特价值
2. **核心链路检查**：确保从论文到 Idea 到 MVE 的完整流程
3. **移动端适配**：确保在手机上可用
4. **动效添加**：卡片 hover、按钮点击等基础动效

### 第二阶段（中优先级）

5. **使用案例丰富**：完善 mock 数据，构建完整案例
6. **未使用组件集成**：将 EvidenceList、PaperCard 等集成到页面
7. **视觉层次优化**：色彩、阴影、间距统一

### 第三阶段（低优先级）

8. **智能推荐引擎展示**：可视化 stateCalculator 和 nextActionCalculator
9. **错误处理**：空状态、边界情况处理
10. **性能优化**：代码分割、懒加载

## 风险与应对

| 风险 | 应对措施 |
|---|---|
| 改动过大导致 build 失败 | 每阶段修改后运行 `npm run build` 验证 |
| 移动端适配复杂度高 | 优先保证核心页面（首页、Dashboard、Papers）的适配 |
| 动效过多影响性能 | 只添加必要的微动画，避免复杂动画 |
| 时间不足无法完成所有改进 | 优先完成第一阶段（高优先级）任务 |
